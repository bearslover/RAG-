问题	                                回答方向
RAG 是什么原理？	        检索 + 生成，解决大模型知识截止问题
为什么用 ChromaDB？	本地持久化，无需部署服务，适合小型项目
MD5去重的作用？	        防止同一文档多次上传重复占用向量库空间
如何实现多轮对话？	        LangChain 的 RunnableWithMessageHistory + 本地 JSON 文件存储历史
项目难点是什么？	        可以说 Prompt 工程调优、流式输出与 Streamlit 组件的 DOM 冲突问题



这个项目实现了以下成果：
技术成果
1. 完整的 RAG 问答链路
从零构建了"文档入库 → 向量检索 → 大模型生成"的完整闭环
用户提问时自动从知识库检索相关内容作为参考，回答更准确、不依赖模型自身训练数据

2. 私有知识库管理
支持通过 Web 页面上传 TXT 文件，自动完成分割、向量化、入库
MD5 去重机制防止重复入库，保证数据一致性

3. 多轮对话记忆
对话历史持久化到本地文件，关闭页面后历史不丢失
支持多用户（多 session_id）隔离存储

4. 流式输出体验
模型回答逐字显示，用户无需等待完整响应，交互体验接近 ChatGPT

5. 两个独立 Web 应用
app_file_uploader.py：知识库维护端
app_qa.py：用户问答端
